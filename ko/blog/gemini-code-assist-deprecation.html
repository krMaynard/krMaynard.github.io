Gemini Code Assist의 GitHub 리뷰어가 오늘 종료됩니다 — 다음은 어디로

2026년 7월 17일
Gemini Code Assist AI 코드 리뷰 개발자 도구 GitHub

Gemini에게 풀 리퀘스트 리뷰를 맡겨 왔다면, 오늘이 바로 그 봇이 조용해지는 날입니다. 2026년 7월 17일부로 구글은 GitHub용 소비자 버전 Gemini Code Assist를 종료했습니다 — 여러분의 PR에 자동화된 코드 리뷰와 요약을 남겨 주던 그 앱 말입니다. 봄부터 단계적으로 진행되어 온 무료·개인 등급 축소의 마지막 절차입니다.

각 요소가 서로 다른 날짜에 꺼졌으므로, 전체 순서를 짚어 보겠습니다.

  • 2026년 6월 18일 — Gemini Code Assist IDE 확장 프로그램과 Gemini CLI가 개인용 Gemini Code Assist 및 Google AI Pro·Ultra 요금제에 대해 요청 처리를 중단했습니다.
  • 2026년 7월 17일 (오늘) — 소비자용 Gemini Code Assist GitHub 앱이 코드 리뷰 활동을 종료합니다.

누구도 성급하게 걱정하기 전에 짚어 둘 중요한 단서 하나 — 이것은 소비자 이야기입니다. 여러분의 조직이 Gemini Code Assist Standard나 Enterprise 라이선스를 유료로 이용하고 있다면, 구글은 그것들이 영향을 받지 않는다고 밝힙니다 — IDE 어시스트, CLI, GitHub 리뷰가 그대로 유지됩니다. 끝나는 것은 많은 1인 개발자와 소규모 팀이 자신의 저장소에 연결해 두었던 무료·개인용 진입로입니다.

구글이 여러분에게 바라는 것

구글이 제시하는 마이그레이션 경로는 5월 I/O에서 공개한 "에이전트 우선" 플랫폼 Antigravity와, 그 터미널 동반 도구인 Antigravity CLI입니다 — Antigravity 2.0 데스크톱 앱과 동일한 에이전트 하니스를 공유하는 Go 재작성판입니다. 5월 19일부터 antigravity.google/download에서 이용할 수 있으며, 여러분의 Agent Skills, Hooks, Subagents, Extensions(이제는 "플러그인")가 그대로 이어집니다. 구글은 출시 시점에 완전한 기능 동등성은 없다고 솔직하게 밝히고 있습니다.

하지만 그 간극에 주목하세요. Antigravity CLI는 여러분이 조종하는 에이전트입니다 — 터미널과 에디터에서 동작하는 대화형 코딩 어시스턴트죠. 오늘 실제로 꺼진 것은 요청하지 않아도 여러분의 GitHub PR을 지켜보고 댓글을 달아 주던 자동 리뷰어였습니다. 이 둘은 같은 일이 아니며, 구글은 소비자용 GitHub 봇을 그대로 대체할 물건을 내놓고 있지 않습니다. 그러니 여러분이 아쉬워할 것이 바로 그 PR 리뷰라면, 솔직한 답은 이렇습니다. Antigravity만으로는 부족하고, 전용 코드 리뷰 도구가 필요할 가능성이 높습니다.

제가 대신 구축한 방식

결국 제 저장소 31개에 걸쳐 작동하는 소규모의 공급업체 중립적 리뷰 대기열을 구성했고, 보호된 32번째 저장소에는 모든 검사를 통과한 풀 리퀘스트로 준비해 두었습니다. GitHub Actions 워크플로가 브랜치 이름, PR 본문 또는 커밋 작성자·세션 메타데이터를 통해 Claude가 작성한 변경 사항을 식별합니다. 해당 풀 리퀘스트가 생성, 업데이트, 재오픈 또는 리뷰 준비 완료되면 codex-review 라벨을 붙입니다. Codex는 이 대기열에서 전체 diff와 해결되지 않은 리뷰 스레드를 읽고, 기존 CI를 확인하고, 관련 로컬 테스트를 실행한 뒤 근거가 포함된 리뷰를 게시하거나 범위를 좁힌 수정 사항을 푸시합니다. 성공적으로 끝나면 대기열 라벨은 codex-reviewed로 바뀌고, 새 커밋이 들어오면 해당 상태가 제거되어 다시 대기열에 등록됩니다. 병합은 여전히 별도의 명시적인 결정입니다.

GitHub App 하나를 설치하는 것보다 구성할 것이 많지만, 제가 실제로 중요하게 생각한 부분은 유지됩니다. 모든 리비전에 대한 자동 리뷰, 눈에 보이는 상태, 재현 가능한 검사, 그리고 코드를 작성한 에이전트가 최종 리뷰까지 맡아야 한다는 가정을 피할 수 있습니다.

실제로 Gemini 리뷰는 얼마나 좋았을까?

앱을 대체하기 전에 풀 리퀘스트 10개를 감사했습니다. 그리고 예정된 종료일에 이 글을 마무리하는 동안 Gemini가 이 PR도 리뷰해 현지화와 SEO에 관한 타당한 지적 14건을 더 남겼습니다. 11건의 리뷰 전체에서 인라인 지적 54건이 나왔고, 높음 5건, 보안-중간 1건, 중간 48건이었습니다. 54건 중 53건은 조치할 가치가 있었고 1건은 오탐으로 판단해 댓글 단위의 유용률은 98.1%였습니다. 여섯 번역에서 같은 메타데이터 필드가 빠진 사례처럼 반복을 묶으면 약 31개의 서로 다른 문제 패턴입니다. 이 마지막 리뷰는 Google이 공표한 소비자용 종료일이 7월 17일인 가운데서도 전환이 즉시 일어나지는 않았음을 보여 줍니다.

유용한 발견은 구체적이었습니다. 누락된 오류 처리, 안전하지 않은 인증서 저장소 변경, Unicode와 로캘에 대한 잘못된 가정, 이식성 없는 systemd 설정, 잘못된 HTML, 예외 시 실행되지 않는 정리 경로, 누락된 도구 사전 조건 등이었습니다. Gemini는 수정 방법이 명확한 국소적 결함에 특히 강했습니다. 반면 중복 통합과 전체 워크플로 추론에는 약점이 있었습니다. 여러 파일이나 언어판에서 같은 근본 원인을 반복 지적했고, 한 번은 노트북 JSON을 Python 튜플로 오해했으며, 이후 2차 리뷰가 발견한 멱등성 경쟁 조건은 놓쳤습니다. 제 평가는 긍정적이지만 한계가 분명합니다. 봇은 실질적인 가치를 많이 만들었지만, 각 의견의 검증과 개별 줄을 넘어 전체를 보는 리뷰어는 여전히 필요했습니다.

요점

구글이 AI 보조 개발을 죽인 것은 아닙니다. 통합한 것입니다. 소비자용 Gemini Code Assist 도구들은 Antigravity로 흡수되고 있고, 엔터프라이즈 라이선스는 변함없이 이어집니다. 하지만 우리 상당수가 기대어 온 그 특정한 편의 — 모든 PR에 조용히 댓글을 달아 주던 무료 리뷰어 — 는 오늘부로 사라졌고, Antigravity는 그것의 곧바른 대체품이 아닙니다. Antigravity로 옮겨 가면 구글의 세계 안에 머무르게 됩니다. 저는 반대 방향으로 가서 PR 리뷰를 Codex에 맡겼고, 그래서 모든 리비전이 여전히 검사를 거칩니다. 어느 쪽이든, 다음 풀 리퀘스트가 리뷰 없이 지나가기 전에 대체재를 연결해 두기에 오늘이 좋은 날입니다.

출처: 구글의 Gemini Code Assist deprecations 페이지(종료 날짜와 엔터프라이즈 예외 조항)와 Gemini CLI → Antigravity CLI 전환 게시글, 그리고 9to5Google의 보도. 제가 마련한 대체재는 OpenAI의 Codex GitHub 연동 위에서 돌아가며, 그것을 구동하는 워크플로는 이 저장소에 공개되어 있습니다.
