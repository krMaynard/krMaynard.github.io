Roblox 투명성 보고 & 컴플라이언스 데이터 모델

Roblox · 2025년~현재
과제

EU DSA, EU TCOR, UK OSA, NY S895, CA AB 587, 브라질 ECA, 인도 IT 규정 등 증가하는 규제들이 각각 세분화되고 일관성 있게 현지화된 투명성 보고를 요구합니다. Roblox의 기존 방식은 수동이었으며, 규제 환경이 계속 확대됨에 따라 확장이 어려웠습니다.

수행 내용
  • 8개 이상의 국제 규제에 대한 투명성 보고 전략 및 프로덕션을 주도합니다.
  • 반복 가능하고 감사 가능한 보고 사이클을 위해 버전 관리된 모듈식 프로덕션급 SQL 쿼리 모음을 작성했습니다.
  • 외부 감사 전 시스템을 확장하고 미래 대비하기 위해 Product Compliance 통합 데이터 모델 및 자동화 보고 파이프라인을 설계 중입니다.
  • 팀 생산성 향상 및 수동 오버헤드 감소를 위한 AI 지원 이니셔티브를 주도합니다.
성과
  • Roblox 역대 가장 포괄적인 EU DSA 투명성 보고서를 발행했습니다 — 콘텐츠 모더레이션, 정부 명령, 불법 콘텐츠, 사용자 이의 신청, 자동화 시스템에 대한 고도로 세분화되고 일관성 있게 현지화된 EU 데이터를 포함한 의무적 통합 보고 형식을 최초로 사용.
  • 투명성 보고의 완전 자동화를 향해 구축 중이며, 수동 오버헤드를 제거하고 확대되는 규제 포트폴리오에 대한 감사 준비성을 향상시킵니다.

Roblox 연구 데이터 API

Roblox · 2025년~현재
과제

온라인 플랫폼을 연구하는 연구자들(학계, 시민 사회 단체, 플랫폼 안전 분석가)은 Roblox의 공개 데이터에 공식적이고 프로그래밍 방식으로 접근할 수 없었습니다. 구조화된 접근 방식 없이는 외부 연구가 수동 수집이나 비공식 스크래핑에 의존해야 했으며, Roblox의 콘텐츠 모더레이션, 플랫폼 안전, 정책 집행에 대한 독립적인 조사의 질과 규모가 제한되었습니다. 한편 Roblox는 주요 플랫폼에 대한 정책 입안자와 공공의 이해를 알리는 연구 커뮤니티를 능동적으로 지원할 수단이 없었습니다.

수행 내용
  • Roblox 최초의 공개 연구 데이터 API를 만들 기회를 발굴했습니다 — 회사에 존재한 적 없는 역량이며, 주요 게임 플랫폼에서 표준 관행이 아닙니다.
  • 엔드투엔드 제품 전략 주도: 공개할 공개 데이터 범위 설정, API 스키마 및 엔드포인트 아키텍처 설계, 데이터 거버넌스·연구자 접근 제어·속도 제한·인증·이용 약관 정의.
  • 엔지니어링, 법무, 정책, 프라이버시 전반의 크로스펑셔널 정렬을 추진하여 컴플라이언트하고 지속 가능하며 확장 가능한 접근 모델을 설계.
  • API를 Roblox의 기존 투명성 보고에 대한 구조화된 보완재로 포지셔닝 — 회사의 공공 책임 체계를 주기적 보고서에서 쿼리 가능한 온디맨드 데이터 접근으로 확장.
성과
  • 연구자들이 Roblox의 공개 플랫폼 데이터에 접근할 수 있는 최초의 공식 프로그래밍 경로를 만들었습니다 — 회사가 이전에 제공한 적 없는 투명성의 중요한 진전.
  • 플랫폼 안전, 콘텐츠 모더레이션 효과성, 정책 집행에 대한 독립적인 조사를 대규모로 가능하게 합니다 — 학계, 시민 사회, 규제 연구 커뮤니티를 직접 지원.
  • 업계 표준을 앞서 주요 게임 및 소셜 플랫폼 중 연구 데이터 접근 분야의 선도 기업으로 Roblox를 포지셔닝.

AI를 활용한 프로덕션 쿼리 개발

Roblox · 2025년~현재
과제

10개 이상의 규제 컴플라이언스 요구사항(보고서, RFI, 위험 평가, 소송 지원)을 위한 프로덕션급 SQL 및 Python 구축에는 일반적으로 사이클당 3~6주의 순수 코딩 엔지니어링이 필요했습니다. 빠르게 확장되는 규제 포트폴리오와 소규모 팀으로는 그 속도가 실행 불가능했습니다.

수행 내용
  • Cursor 에이전트를 사용하여 프로덕션급 SQL 및 Python을 생성하는 인간 참여형(human-in-the-loop) AI 코딩 워크플로에서 PM 겸 초안 작성자로 활동.
  • 10개 이상의 규제 요구사항에 걸쳐 7,000줄 이상의 프로덕션 코드를 생성했으며, 데이터 엔지니어와 데이터 사이언티스트가 주요 코더가 아닌 코드 리뷰어 역할을 담당.
  • 구조화된 개발 파이프라인 설계: 쿼리 및 스키마 컨텍스트 → AI 생성 SQL → 인간 코드 리뷰 및 승인 → 반복 개선 → 풀스케일 Python 파이프라인.
  • PM과 엔지니어링 사이의 피드백 루프를 단축하여 더 빠른 반복과 크로스펑셔널 일정 조율 병목 감소.
성과
  • 코딩 3~6 엔지니어링 주를 수시간 또는 수일의 AI 가이드 작업으로 압축.
  • 4,800만 토큰 이상의 프로덕션 코드를 총 비용 $77에 생성.
  • 헤드카운트를 선형으로 늘리지 않고 10개 이상의 규제 요구사항에 걸쳐 코드 생산을 확장.
  • PM이 로직을 작성하고 엔지니어가 결과물을 검토하는 재현 가능한 인간 참여형 모델 확립.
Cursor AI 코딩 SQL Python 데이터 파이프라인 컴플라이언스 프로세스 자동화

Sandpiper — Privacy Sandbox AI 지식 어시스턴트

Google · 2025년
과제

Privacy Sandbox 워킹 그룹(900명 이상의 엔지니어, PM, 컴플라이언스 직원)은 제한된 리포지토리에 보관된 12,000개 이상의 문서에 파묻혀 있었습니다. 정보 검색이 심각한 병목 현상이 되어 개발, 규제 컴플라이언스, 전략을 지연시켰지만, 코퍼스의 민감성 때문에 범용 검색 도구 사용이 불가능했습니다.

수행 내용
  • v1 및 v2 설계 문서의 단독 작성자; 구상에서 작동 프로토타입까지의 주요 추진자.
  • 조직 전반의 이해관계자들을 컨설팅하여 페인 포인트를 진단하고, 완전한 엔드투엔드 시스템을 설계: 웹 프론트엔드, Agent Studio 기반 백엔드, LLM으로 Gemini 2.5 Pro, 12,000개 이상의 제한 문서로 인덱싱된 벡터 데이터베이스.
  • 두 가지 운영 모드 구현: (1) 코퍼스 기반 RAG 및 출처 인용이 있는 전문가 Q&A, (2) 사용자 제공 변경 설명에서 D&I 기준 평가 초안을 직접 생성하는 고빈도 컴플라이언스 워크플로를 위한 특화 초안 모드.
  • 시스템 프롬프트 작성, 기술 요구사항, 접근 제어, 환각 방지 및 필수 인간 검토 등 위험 완화 정의.
  • Privacy Sandbox TPgM AI 해커톤에서 프로젝트 발표; ‘시간 절약 및 규제 대응 품질 향상 측면에서 잠재적 영향력을 열었다’는 이유로 시니어 디렉터로부터 스팟 보너스 수상.
성과
  • Google Privacy Sandbox TPgM AI 해커톤에서 2위 수상.
  • Privacy Sandbox 워킹 그룹 전체의 900명 이상 사용자에게 서비스하도록 설계.
  • 정보 검색 및 초기 컴플라이언스 문서 초안 작성에 소요되는 시간을 크게 단축.
  • 컴플라이언스 제약(통제된 문서 코퍼스)을 지식 관리 이점으로 전환.
RAG Gemini 2.5 Pro 벡터 데이터베이스 Agent Studio 시스템 설계 프롬프트 엔지니어링 컴플라이언스

GenAI 법적 검토 자동화

Google · 2024~2025년
과제

Privacy Sandbox 법무팀은 매주 몇 시간씩 버그 보고서와 연결된 문서를 수동으로 검토하여 법적 검토 다이제스트를 준비했습니다. 이 프로세스는 느리고 일관성이 없었으며, Privacy Sandbox 전반의 컴플라이언스 감독에 병목이 되었습니다.

수행 내용
  • 엔드투엔드 자동화 솔루션의 단독 작성자 및 개발자.
  • 법무팀과 직접 협업하여 워크플로, 의사결정 기준, 각 하위 팀에서의 ‘우수한 결과’가 무엇인지 파악.
  • 3단계 파이프라인 설계: SQL(F1)로 관련 버그 데이터 추출 → Python(Colab)으로 조율 → TextMiner를 통해 Gemini LLM이 처리.
  • 복잡한 인지 작업을 수행하는 다단계 프롬프트 엔지니어링: 연결된 문서 요약, 특정 CMA 규제 약정에 대한 주요 위험 식별, 세 법무 하위 팀(Product, Competition, Privacy)에 대한 검토 유형(‘검토 필요’ vs ‘참고만’) 분류.
성과
  • Privacy Sandbox 전체의 프로그램 매니저들을 위해 주당 수 시간의 수동 작업을 절약.
  • 매 사이클마다 일관성 있고 시기 적절하며 고품질의 다이제스트를 법무에 제공 — 중요 항목 누락 위험 감소.
  • 법적 검토 사이클 가속화 및 컴플라이언스 감독의 전반적인 견고성 향상.

MT 컴플라이언스 보고 자동화

Google · 2024~2025년
과제

영국 경쟁시장청(CMA)과의 법적 약정에 따른 필수 산출물인 분기별 Monitoring Trustee(MT) 컴플라이언스 보고서가 수동으로 생성되었습니다. 이 프로세스는 분기당 60~80 인시 이상이 소요되었으며, 고위험·오류 발생·문서화 부재 상태였고, 5~6명의 개인 단일 장애점에 의존했습니다. 감사 가능성도 없었고 주요 지표에 대한 일별 가시성도 없었습니다.

수행 내용
  • 단독 작성자 및 구현자. 원래 프로젝트를 스코핑한 세 명의 SWE는 소프트웨어 엔지니어만 구축할 수 있다고 믿었으며 TPM이 시도하는 것에 회의적이었습니다.
  • 엔드투엔드 시스템 아키텍처 설계: 여러 내부 데이터 소스(Teamgraph, 원시 시스템 로그, 비표준 임포트)에서 데이터를 가져오는 ETL 파이프라인, 복잡한 데이터 처리, LLM 보강, 자동 보고서 생성.

과제

LLM 케이스 요약 대시보드

Google · 2023~2024년
과제

Legal Content Policy & Standards팀은 케이스 관리 및 트렌드 분석을 위해 수천 건의 법적 삭제 요청의 본질을 빠르게 파악해야 했습니다. 이 규모에서의 수동 검토는 불가능했습니다. 자동 케이스 요약은 주요 관리 대시보드에서 가장 많이 요청된 기능이었지만 확장 가능한 솔루션이 존재하지 않았습니다.

수행 내용
  • 설계 문서, 사용자 가이드 작성자, 완전한 자동화 워크플로의 주요 구현자.
  • LCPS팀과 직접 협의하여 요구사항을 정의하고 케이스 요약이 운영적으로 어떻게 사용될지 뉘앙스 파악.
  • SQLMiner를 사용하여 데이터베이스 환경 내에서 직접 대규모 LLM 추론을 수행하는 혁신적인 파이프라인 설계 — 민감한 케이스 데이터를 중간 시스템으로 내보내지 않고도 진정한 산업 규모 운영 가능.
  • 케이스 테이블의 구조화된 데이터를 LLM에 공급하는 RAG 패턴 프롬프트를 엔지니어링하여 모든 대상 케이스에 걸쳐 간결하고 정확하며 일관된 형식의 요약 생성.
  • AI 생성 요약을 팀의 주요 PLX 관리 대시보드에 온디맨드 기능으로 직접 표시.
성과
  • 전체 LCPS팀의 일상적인 케이스 관리에서 상당한 효율성 향상.
  • 법적 삭제 케이스 코퍼스 전반의 트렌드 분석 속도 및 정확도 향상.
  • 팀 포커스를 저수준 개별 케이스 검토에서 고수준 전략적 감독으로 전환.

Google · 2021~2023년
과제
