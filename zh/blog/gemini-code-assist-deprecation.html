Gemini Code Assist的GitHub代码审查今天关停——接下来该去哪里

2026年7月17日
Gemini Code Assist AI代码审查 开发者工具 GitHub

如果你一直让Gemini来审查你的拉取请求（pull request），那么今天就是这个机器人归于沉默的日子。自2026年7月17日起，Google已关闭面向消费者的Gemini Code Assist在GitHub上的版本——也就是那个在你的PR上自动发布代码审查与摘要的应用。这是自今春以来分阶段收尾免费版与个人版的最后一步。

下面是完整的时间线，因为各个部分是在不同日期关停的：

  • 2026年6月18日——Gemini Code Assist的IDE扩展和Gemini CLI针对个人版Gemini Code Assist以及Google AI Pro和Ultra套餐，停止响应请求。
  • 2026年7月17日（今天）——面向消费者的Gemini Code Assist GitHub应用终止其代码审查活动。

在有人惊慌之前，有一点重要的提醒：这说的是消费者版的情况。如果你所在的组织购买了Gemini Code Assist的Standard或Enterprise许可，Google表示这些不受影响——你的IDE辅助、CLI和GitHub审查会继续运行。要终止的是那个免费/个人版的入门通道，而许多独立开发者和小团队都把它接入了自己的代码仓库。

Google希望你怎么做

Google自己给出的迁移路径是Antigravity——它在5月I/O大会上发布的“以智能体为先”（agent-first）的平台——外加它的终端搭档Antigravity CLI，这是一个用Go语言重写的版本，与Antigravity 2.0桌面应用共享同一套智能体框架。它自5月19日起可在antigravity.google/download获取，你的Agent Skills、Hooks、Subagents和Extensions（现称“plugins”）都可以延续使用。Google坦言，在发布之初并不具备完整的功能对等。

但请注意其中的落差。Antigravity CLI是一个由你驱动的智能体——你终端和编辑器里的一个交互式编码助手。而今天真正关停的那个东西，是一个盯着你GitHub PR、无需你开口就自动评论的自动审查器。这两者做的不是同一件事，而且Google并没有为消费者版的GitHub机器人推出一个一对一的替代品。所以，如果你会怀念的是那种PR审查，那么诚实的答案是：你八成需要的是一款专门的代码审查工具，而不只是Antigravity。

面向GitHub PR审查这项工作的替代方案

好消息是——这正是当下开发者工具市场里最拥挤的角落之一。大致按你需要改动的多少从少到多排列：

  • GitHub Copilot code review——如果你已经在为Copilot付费，这是阻力最小的路径。它原生于GitHub，无需任何新的集成，并在摘要之下叠加了CodeQL和ESLint扫描。比那些专门工具浅一些，但开启起来即刻可用。
  • CodeRabbit——最受欢迎的专门AI审查器，直接在PR里发布结构化的逐行反馈。付费档位起价约为每位开发者每月24美元，并为开源和公开仓库提供免费档位——是最接近替代Gemini机器人所做之事的现成之选。
  • Qodo Merge（原名Codium/PR-Agent）——提供PR摘要、审查和内联建议，面向那些同时还想要测试生成与治理能力的团队。其PR-Agent核心是开源的，因此如果你不愿把自己的代码差异交给又一个SaaS，也可以自行托管。
  • Greptile——依托的是深度的、整个代码库范围的上下文，而不仅仅是代码差异；在捕捉那些只有放在更大仓库语境中才说得通的问题上很强。约为每位开发者每月30美元。
  • Claude Code——如果你更愿意在CI中运行一个智能体，它的GitHub Action可以审查PR（也能创建PR），因此你可以精确地脚本化要检查的内容。比一个托管的机器人需要更多设置，但工作流由你掌控。（如实相告：本文就是用它起草的。）

如果你偏爱的恰恰是Google的模型，那么请注意，上述其中几款都支持Gemini——你不必为了放弃这个机器人而放弃这个模型。

我的替代方案

我最终为自己的31个代码库搭建了一个小型、供应商中立的审查队列；受保护的第32个代码库已通过一个检查全部通过的拉取请求做好准备。一个GitHub Actions工作流会通过分支名、PR正文或提交作者与会话元数据识别Claude创建的更改。当相应拉取请求被打开、更新、重新打开或标记为可供审查时，它会添加codex-review标签。Codex从队列中读取完整差异和未解决的审查线程，检查现有CI，运行相关的本地测试，并发布有证据支持的审查意见或推送范围明确的修复。成功完成后，队列标签会替换为codex-reviewed；新的提交会取消这一状态并重新排队。合并仍然是一个独立且必须明确作出的决定。

这比安装单个GitHub App需要更多配置，但保留了我真正重视的部分：每次修订都自动审查、状态清晰可见、检查可以复现，而且不必默认让编写代码的代理同时充当最终审查者。

Gemini的实际表现如何？

在替换这个应用之前，我审计了10个拉取请求。随后，在预定关停日完成本文时，Gemini也审查了这篇文章的PR，又提出14条有效的本地化和SEO意见。11次审查共发布了54条行内意见：5条高优先级、1条安全中优先级、48条中优先级。我判断其中53条值得采取行动，1条为误报；按评论计算，有用率为98.1%。合并同一问题的重复实例——例如六种译文缺少同一个元数据字段——后，约对应31种不同的问题模式。这次最后的审查表明，尽管Google公布的消费者版关停日期仍是7月17日，切换并非瞬间完成。

有用的发现都很具体：缺少错误处理、不安全地修改证书存储、对Unicode和区域设置的错误假设、不可移植的systemd配置、格式错误的HTML、异常发生时失效的清理路径，以及遗漏的工具依赖检查。Gemini尤其擅长发现修复方式明确的局部缺陷，但在归并重复问题和理解完整工作流方面较弱：有些评论在不同文件或语言版本中重复同一根因；它曾把笔记本JSON误读成Python元组；也没有发现后来由第二轮审查找出的幂等性竞态。我的结论总体正面但有边界：机器人确实创造了大量价值，但每条意见仍需验证，也仍需要能够跳出单行代码审视全局的审查者。

要点小结

Google并没有扼杀AI辅助开发；它把它整合了。面向消费者的Gemini Code Assist工具正被并入Antigravity，而企业许可则原封不动地继续运行。但很多人所依赖的那份具体便利——一个默默为每个PR评论的免费审查器——从今天起就没有了，而Antigravity并不是它的直接替代。迁移到Antigravity能让你继续留在Google的世界里；而选用Copilot的审查器、CodeRabbit、Qodo或Greptile，则能把你的PR审查找回来。无论选哪条路，今天都是把替代方案接好的好日子——趁下一个拉取请求还没在无人审查中溜走之前。
