五十份通知对一份：代码在 GitHub 上究竟是如何被下架的

July 14, 2026
透明度 数据

2021—2025 年间，GitHub 报告处理了 10,687 份 DMCA 下架通知，而全球所有政府加起来的下架请求仅有 204 份，其中 88% 来自同一个国家：俄罗斯。

核心结论：GitHub 的透明度数据显示，这个全球最大代码托管平台上的内容治理，压倒性地是一套版权制度，而非政府审查制度——在 2021—2025 年间，DMCA 通知数量以大约 50 比 1 的比例超过政府下架请求，即便在政府需求达到峰值的 2025 年也有 25 比 1。最大的保留在于构成：这个「政府」序列其实是一个俄罗斯序列（204 份请求中有 180 份），而 GitHub 报告称几乎没有回应俄罗斯近期的任何要求——2025 年是 97 份中的 3 份——因此政府所要求的与代码实际遭遇之间的落差，比请求数字所暗示的还要大。

  • 数据：github_metrics，Transparency Report API，2014—2025 日历年（多数序列为 2021—2025 年；国家安全区间为 2014—2025 年）。

真正在运转的治理制度是版权

GitHub 的政府下架管道微不足道。而它的 DMCA 管道则是一套工业化流程：每年约处理两千份通知，反通知、撤回和撤销仅追回其中约 2%（五年内 240 份，对应 10,687 份通知）。与此同时，增长最快的版权类别是第 1201 条反规避主张——yt-dlp 和 Twitch 工具争议背后的法律依据——GitHub 报告称其从 2021 年的 92 项升至 2025 年的 645 项，增长七倍（而 2015 年仅有 18 项）。

如果你想弄清楚是什么让代码从 GitHub 上被移除，真正的战场是 DMCA 案卷——而非任何政府部门。

俄罗斯要求得越来越多，得到的却越来越少

五年间，总共只有十一个政府出现在下架数据中（俄罗斯、美国、中国、印度，以及新加坡和斯里兰卡等少数一次性的请求方）。俄罗斯占了 204 份请求中的 180 份，而它的轨迹就是整个故事：请求量从 2021 到 2025 年大约翻了五倍，而 GitHub 的执行率却从 89% 崩塌至 3%。

年份 收到的俄罗斯请求 已执行的俄罗斯请求 执行率
2021 19 17 89%
2022 11 5 45%
2023 11 8 73%
2024 42 13 31%
2025 97 3 3%

执行的依据也变了。2021 年，GitHub 报告称依据「当地法律」处理了 20 份请求（其中 17 份来自俄罗斯，在境内对内容进行地域封锁）。从 2022 到 2025 年，当地法律项下的数量每年都是零：GitHub 报告称唯一予以执行的政府请求，都是那些被标记内容本身就已独立违反其自身服务条款的情形。换句话说，GitHub 的数据表明，它在 2021 年之后彻底停止了履行合法合规性下架——政府如今只有在指出 GitHub 无论如何都会移除的内容时才会「得逞」。

更安静的压力：用户数据，而非下架

政府从 GitHub 那里作为证据保管库所得到的，远多于作为审查者所得到的。2025 年，GitHub 报告收到 650 份用户信息请求（以刑事传票居多，达 385 份）并作出 517 次披露。而以区间呈现的国家安全序列显示出明显的攀升：受国家安全信函与命令影响的账户数量，从 2014 年到 2022 年初每半年都处于 0—249 区间，随后稳步上升，至 2025 年上半年达到 2,000—2,249。信函／命令的数量几乎始终停留在最低区间（2024 年末曾出现一次跃升至 750—999），这意味着更少的手段却卷入了多得多的账户。

保留说明。所有数字都是 GitHub 自行报告的统计；「GitHub 报告」这一措辞在全文中都在发挥实质作用，尤其是对于它在 2025 年回应了俄罗斯 97 份请求中 3 份这一说法（年末收到的请求可能会在稍后解决，尽管此前没有哪一年显示出如此之大的滞后）。政府序列由俄罗斯主导——排除俄罗斯后，其他所有政府五年内合计只发出 24 份请求——因此「政府下架」的趋势实际上就是俄罗斯的趋势。国家安全和 EU DSA 的数字是区间范围（美国政府强制规定的区间），而非精确统计，且不能相加；我只比较区间，绝不求和。DMCA「已处理的通知」统计的是通知，而非代码仓库——单一份通知可能影响许多分支（fork）。最新数据年份：年度序列为 2025 日历年，国家安全区间为 2025 年 7—12 月。

数据：github_metrics，Transparency Report API，2014—2025 年（年度序列为 2021—2025 年）。
