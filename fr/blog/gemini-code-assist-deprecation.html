Le relecteur GitHub de Gemini Code Assist s'éteint aujourd'hui — où aller ensuite

17 juillet 2026
Gemini Code Assist Revue de code par IA Outils pour développeurs GitHub

Si vous laissiez Gemini relire vos pull requests, c'est aujourd'hui que le bot se tait. Depuis le 17 juillet 2026, Google a désactivé la version grand public de Gemini Code Assist sur GitHub — l'application qui publiait des revues de code et des résumés automatisés sur vos PR. C'est la dernière étape d'un arrêt échelonné des offres gratuite et individuelle, en cours de déploiement depuis le printemps.

Voici la séquence complète, car les différents éléments se sont éteints à des dates distinctes :

  • 18 juin 2026 — les extensions IDE de Gemini Code Assist et la Gemini CLI ont cessé de traiter les requêtes pour Gemini Code Assist pour les particuliers ainsi que pour les forfaits Google AI Pro et Ultra.
  • 17 juillet 2026 (aujourd'hui) — l'application GitHub grand public Gemini Code Assist met fin à son activité de revue de code.

Une précaution importante avant que quiconque ne panique : il s'agit ici du volet grand public. Si votre organisation paie une licence Gemini Code Assist Standard ou Enterprise, Google indique que celles-ci sont non concernées — votre assistance dans l'IDE, la CLI et les revues GitHub continuent de fonctionner. Ce qui prend fin, c'est la porte d'entrée gratuite/individuelle que beaucoup de développeurs indépendants et de petites équipes avaient branchée sur leurs dépôts.

Ce que Google veut que vous fassiez

Le chemin de migration proposé par Google est Antigravity, la plateforme « agent-first » dévoilée à I/O en mai, accompagnée de son pendant en terminal, Antigravity CLI — une réécriture en Go qui partage le même harnais d'agent que l'application de bureau Antigravity 2.0. Elle est disponible depuis le 19 mai sur antigravity.google/download, et vos Agent Skills, Hooks, Subagents et Extensions (désormais « plugins ») sont repris. Google le dit franchement : il n'y a pas de parité fonctionnelle complète au lancement.

Mais notez l'écart. Antigravity CLI est un agent que vous pilotez — un assistant de codage interactif dans votre terminal et votre éditeur. Ce qui s'est réellement éteint aujourd'hui, c'était un relecteur automatisé qui surveillait vos PR GitHub et les commentait sans qu'on le lui demande. Ce ne sont pas les mêmes tâches, et Google ne livre pas de remplaçant équivalent pour le bot GitHub grand public. Donc si ce qui va vous manquer, ce sont les revues de PR, la réponse honnête est : il vous faut probablement un outil de revue de code dédié, et pas seulement Antigravity.

Ce que j'ai mis en place à la place

J'ai finalement assemblé une petite file de revue indépendante des fournisseurs pour 31 de mes dépôts, le 32e dépôt protégé étant prêt dans une pull request dont tous les contrôles sont au vert. Un workflow GitHub Actions reconnaît la contribution de Claude grâce au nom de branche, au corps de la PR ou aux métadonnées d'auteur et de session des commits. Lorsqu'une de ces pull requests est ouverte, mise à jour, rouverte ou déclarée prête à être relue, il ajoute le libellé codex-review. Codex prend la file en charge, lit le diff complet et les fils de revue non résolus, vérifie la CI existante, exécute les tests locaux pertinents et publie soit une revue étayée, soit une correction strictement ciblée. Une passe réussie remplace le libellé par codex-reviewed ; tout nouveau commit annule cet état et remet la PR dans la file. La fusion reste une décision distincte et explicite.

Cela demande davantage de plomberie que l'installation d'une seule GitHub App, mais conserve ce qui comptait vraiment pour moi : une revue automatique à chaque révision, un état visible, des vérifications reproductibles et aucune obligation de confier la revue finale à l'agent qui a écrit le code.

Quelle était réellement la qualité de Gemini ?

Avant de remplacer l'application, j'ai audité 10 pull requests. Pendant que je terminais cet article à la date d'arrêt prévue, Gemini a aussi examiné cette PR et ajouté 14 remarques valides de localisation et de SEO. Sur les 11 revues, Gemini a publié 54 remarques en ligne : 5 de priorité haute, 1 sécurité-moyenne et 48 moyennes. J'en ai jugé 53 exploitables et une comme faux positif, soit un taux d'utilité de 98,1 % par commentaire. Après regroupement des répétitions — tel un champ de métadonnées absent dans six traductions — elles représentent environ 31 types de problèmes distincts. Cette dernière revue montre que la coupure n'a pas été instantanée, même si la date d'arrêt grand public publiée par Google restait le 17 juillet.

Les constats utiles étaient concrets : gestion d'erreurs absente, modification dangereuse du magasin de certificats, hypothèses erronées sur Unicode et la locale, configuration systemd non portable, HTML mal formé, nettoyage défaillant lors d'exceptions et prérequis d'outils manquants. Gemini était particulièrement bon sur les défauts locaux assortis d'une correction claire. Il l'était moins pour regrouper les doublons et raisonner sur l'ensemble du workflow : plusieurs commentaires répétaient la même cause dans différents fichiers ou langues, il a une fois confondu du JSON de notebook avec un tuple Python, et il n'a pas repéré une condition de concurrence d'idempotence trouvée lors d'une seconde revue. Mon bilan est favorable mais nuancé : le bot a apporté beaucoup de valeur réelle, mais ses remarques devaient toujours être vérifiées par une personne capable de regarder au-delà d'une ligne isolée.

Ce qu'il faut en retenir

Google n'a pas tué le développement assisté par IA ; il l'a consolidé. Les outils grand public Gemini Code Assist sont intégrés à Antigravity, et les licences entreprise poursuivent leur cours sans changement. Mais la commodité précise sur laquelle beaucoup d'entre nous s'appuyaient — un relecteur gratuit qui commentait discrètement chaque PR — a disparu depuis aujourd'hui, et Antigravity n'en est pas un remplacement direct. Migrer vers Antigravity vous garde dans l'univers de Google ; j'ai pris l'autre chemin et confié la revue des PR à Codex, de sorte que chaque révision reste vérifiée. Quoi qu'il en soit, aujourd'hui est un bon jour pour brancher le remplaçant avant que la prochaine pull request ne passe sans relecture.
