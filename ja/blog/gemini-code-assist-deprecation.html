Gemini Code AssistのGitHubレビュアーが本日停止——次にどこへ行くべきか

2026年7月17日
Gemini Code Assist AIコードレビュー 開発者ツール GitHub

Geminiにプルリクエストをレビューさせていたなら、そのボットが沈黙するのは本日だ。2026年7月17日をもって、GoogleはGitHub上のコンシューマー版Gemini Code Assistを停止した——あなたのPRに自動化されたコードレビューと要約を投稿していたアプリだ。これは、この春から段階的に展開されてきた無料および個人向けティアの終了プロセスの、最後のステップにあたる。

各機能が別々の日付で停止したため、全体の流れを示しておこう。

  • 2026年6月18日——Gemini Code AssistのIDE拡張機能とGemini CLIが、個人向けおよびGoogle AI Pro・Ultraプラン向けのGemini Code Assistについてリクエストの処理を停止した。
  • 2026年7月17日（本日）——コンシューマー版Gemini Code AssistのGitHubアプリがコードレビュー活動を終了する。

誰かがパニックになる前に、一つ重要な但し書きを。これはあくまでコンシューマー向けの話だ。もしあなたの組織がGemini Code AssistのStandardまたはEnterpriseライセンスを購入しているなら、Googleはそれらは影響を受けないとしている——IDE支援、CLI、GitHubレビューはそのまま動き続ける。終わるのは、多くの個人開発者や小規模チームが自分たちのリポジトリに組み込んでいた、無料/個人向けの入り口だ。

Googleが望んでいること

Google自身が示す移行先はAntigravity——5月のI/Oで発表された「エージェントファースト」のプラットフォーム——と、そのターミナル用の相棒であるAntigravity CLIだ。後者はGoで書き直されたもので、デスクトップアプリのAntigravity 2.0と同じエージェントハーネスを共有する。5月19日からantigravity.google/downloadで利用可能になっており、あなたのAgent Skills、Hooks、Subagents、そしてExtensions（現在は「plugins」）はそのまま引き継がれる。Googleは、ローンチ時点では完全な機能パリティはないことを率直に認めている。

だが、ここにギャップがあることに注目してほしい。Antigravity CLIはあなたが操作するエージェントだ——ターミナルとエディタの中にある対話的なコーディングアシスタントである。本日実際に切れたものは、あなたのGitHub PRを監視し、頼まれもしないのにコメントを付けていた自動レビュアーだった。この二つは同じ仕事ではないし、Googleはコンシューマー版のGitHubボットに対する同等の置き換えを出すつもりはない。だから、あなたが恋しく思うのがPRレビューだとすれば、正直な答えはこうだ——おそらくAntigravityだけでなく、専用のコードレビューツールが必要になる。

GitHub PRレビューという仕事のための代替手段

朗報だ——ここは今、開発者ツール市場のなかでも最も混み合った一角の一つだ。変更の手間が少ない順に、おおよそ並べてみよう。

  • GitHub Copilot code review——すでにCopilotに料金を払っているなら、最も抵抗の少ない道。GitHubにネイティブで、新たな連携は一切不要、要約の下にCodeQLとESLintのスキャンを重ねてくれる。専用ツールより浅いが、オンにするのは一瞬だ。
  • CodeRabbit——最も人気の専用AIレビュアーで、構造化された行単位のフィードバックをPRに直接投稿する。有料ティアは開発者1人あたり月24ドル前後から、オープンソースや公開リポジトリ向けの無料ティアもある——Geminiボットがやっていたことに最も近いドロップイン。
  • Qodo Merge（旧Codium/PR-Agent）——PRの要約、レビュー、インラインの提案を提供し、テスト生成やガバナンスも求めるチームを想定している。中核のPR-Agentはオープンソースなので、自分の差分を別のSaaSに渡したくなければセルフホストできる。
  • Greptile——差分だけでなく、コードベース全体の深いコンテキストに依拠する。より広いリポジトリの文脈でしか意味をなさない問題を捉えるのに強い。開発者1人あたり月30ドル前後。
  • Claude Code——むしろCIの中でエージェントを走らせたいなら、そのGitHub ActionがPRをレビューし（PRを起こすこともできる）、何をチェックするかを正確にスクリプト化できる。ホスト型のボットより設定の手間はかかるが、ワークフローは自分のものになる。（正直に開示しておくと、この記事はこれを使って草稿を書いた。）

もしあなたの忠誠がGoogleのモデルそのものに向いているなら、Geminiはこれらのいくつかを通じて利用できることに留意してほしい——ボットを手放すために、モデルまで手放す必要はない。

私が代わりに構築したもの

最終的に、20のリポジトリを対象とする小規模でベンダー中立のレビューキューを組みました。GitHub Actionsワークフローが、Claude作成のプルリクエストのオープン、更新、再オープン、またはレビュー準備完了を検知し、codex-reviewラベルを付けます。Codexはそのキューから、差分全体と未解決のレビュースレッドを読み、既存のCIを確認し、関連するローカルテストを実行します。そのうえで、根拠を示したレビューを投稿するか、範囲を限定した修正をプッシュします。レビューが完了するとラベルはcodex-reviewedに変わり、新しいコミットが入ると再びレビュー待ちになります。マージは引き続き、明示的に別途判断します。

単一のGitHub Appを入れるより配管は増えますが、私が本当に必要としていた部分は保てます。つまり、各リビジョンの自動レビュー、見える状態、再現可能なチェック、そしてコードを書いたエージェントを最終レビュアーにもする必要がないことです。

まとめ

GoogleはAI支援の開発を殺したわけではない。統合したのだ。コンシューマー版のGemini Code AssistツールはAntigravityに畳み込まれつつあり、エンタープライズライセンスは変わらず続く。しかし、私たちの多くが頼っていた特定の便利さ——あらゆるPRに静かにコメントしてくれる無料のレビュアー——は本日をもって消え、Antigravityはそれの素直な置き換えではない。Antigravityへの移行はあなたをGoogleの世界にとどめてくれる。Copilotのレビュアー、CodeRabbit、Qodo、Greptileを導入すれば、PRレビューが戻ってくる。いずれにせよ、次のプルリクエストがレビューされないまま流れていく前に、代替を組み込んでおくには本日がうってつけの日だ。
