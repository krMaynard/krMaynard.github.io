Il reviewer GitHub di Gemini Code Assist si spegne oggi — dove andare adesso

17 luglio 2026
Gemini Code Assist AI Code Review Developer Tools GitHub

Se lasci che Gemini riveda le tue pull request, oggi è il giorno in cui il bot ammutolisce. A partire dal 17 luglio 2026, Google ha spento la versione consumer di Gemini Code Assist su GitHub — l'app che pubblicava revisioni del codice e riepiloghi automatici sulle tue PR. È l'ultimo passo di una dismissione a tappe dei piani gratuito e individuale in corso dalla primavera.

Ecco la sequenza completa, perché i pezzi si sono spenti in date diverse:

  • 18 giugno 2026 — le estensioni IDE di Gemini Code Assist e la Gemini CLI hanno smesso di servire richieste per Gemini Code Assist per privati e per i piani Google AI Pro e Ultra.
  • 17 luglio 2026 (oggi) — l'app GitHub consumer di Gemini Code Assist cessa la sua attività di revisione del codice.

Una precisazione importante prima che qualcuno vada nel panico: questa è la storia consumer. Se la tua organizzazione paga una licenza Gemini Code Assist Standard o Enterprise, Google dice che quelle sono non interessate — l'assist nell'IDE, la CLI e le revisioni su GitHub continuano a funzionare. Ciò che finisce è la porta d'accesso gratuita/individuale che molti sviluppatori solitari e piccoli team avevano collegato ai propri repository.

Cosa Google vuole che tu faccia

Il percorso di migrazione proposto da Google è Antigravity, la piattaforma «agent-first» presentata all'I/O di maggio, più il suo compagno da terminale, Antigravity CLI — una riscrittura in Go che condivide lo stesso agent harness dell'app desktop Antigravity 2.0. È disponibile dal 19 maggio su antigravity.google/download, e i tuoi Agent Skills, Hooks, Subagents ed Extensions (ora «plugin») vengono trasferiti. Google è chiara sul fatto che al lancio non c'è piena parità di funzionalità.

Ma nota il divario. Antigravity CLI è un agente che guidi tu — un assistente di coding interattivo nel tuo terminale e nel tuo editor. La cosa che oggi si è effettivamente spenta era un reviewer automatico che sorvegliava le tue PR su GitHub e le commentava senza che glielo si chiedesse. Non sono lo stesso lavoro, e Google non sta rilasciando un sostituto identico per il bot GitHub consumer. Quindi se ciò che ti mancherà sono le revisioni delle PR, la risposta onesta è: probabilmente ti serve uno strumento dedicato di code review, non solo Antigravity.

Alternative per il lavoro di revisione delle PR su GitHub

Buone notizie — questo è oggi uno degli angoli più affollati del mercato degli strumenti per sviluppatori. All'incirca in ordine di quanto poco dovresti cambiare:

  • GitHub Copilot code review — la via di minor resistenza se già paghi Copilot. È nativo di GitHub, non richiede alcuna nuova integrazione e affianca ai riepiloghi la scansione con CodeQL ed ESLint. Più superficiale degli strumenti dedicati, ma si attiva all'istante.
  • CodeRabbit — il reviewer AI dedicato più diffuso, che pubblica feedback strutturati riga per riga direttamente nella PR. I piani a pagamento partono da circa 24 $/sviluppatore/mese, con un piano gratuito per progetti open-source e repository pubblici — il sostituto più immediato di ciò che faceva il bot Gemini.
  • Qodo Merge (già Codium/PR-Agent) — riepiloghi delle PR, revisione e suggerimenti inline, pensato per team che vogliono anche generazione di test e governance. Il suo nucleo PR-Agent è open source, quindi puoi fare self-hosting se preferisci non affidare i tuoi diff a un altro SaaS.
  • Greptile — si appoggia a un contesto profondo dell'intero codebase anziché al solo diff; forte nel cogliere problemi che hanno senso solo nel contesto del repository più ampio. Intorno ai 30 $/sviluppatore/mese.
  • Claude Code — se preferisci far girare un agente in CI, la sua GitHub Action rivede le PR (e può aprirle), così definisci via script esattamente cosa viene controllato. Più configurazione di un bot ospitato, ma il workflow è tuo. (Per trasparenza: questo articolo è stato redatto con esso.)

Se la tua fedeltà è specificamente ai modelli di Google, nota che Gemini è disponibile attraverso molti di questi — non devi rinunciare al modello per rinunciare al bot.

Con cosa l'ho sostituito

Alla fine ho configurato una piccola coda di revisione indipendente dal fornitore per più di 20 dei miei repository. Un workflow GitHub Actions riconosce la paternità di Claude dal nome del branch, dal corpo della PR o dai metadati di autore e sessione dei commit. Quando una di queste pull request viene aperta, aggiornata, riaperta o contrassegnata come pronta per la revisione, aggiunge l'etichetta codex-review. Codex prende in carico la coda, legge l'intero diff e i thread di revisione irrisolti, controlla la CI esistente, esegue i test locali pertinenti e pubblica una revisione supportata da evidenze oppure invia una correzione strettamente circoscritta. Al termine di una revisione riuscita, l'etichetta viene sostituita da codex-reviewed; un nuovo commit annulla quello stato e rimette la PR in coda. Il merge resta una decisione separata ed esplicita.

Richiede più configurazione rispetto all'installazione di una singola GitHub App, ma conserva ciò che per me contava davvero: revisione automatica a ogni modifica, stato visibile, controlli riproducibili e nessuna presunzione che l'agente che scrive il codice debba esserne anche il revisore finale.

La conclusione

Google non ha ucciso lo sviluppo assistito dall'IA; lo ha consolidato. Gli strumenti consumer di Gemini Code Assist vengono confluiti in Antigravity, e le licenze enterprise proseguono invariate. Ma la comodità specifica su cui molti di noi contavano — un reviewer gratuito che commentava discretamente ogni PR — da oggi è sparita, e Antigravity non ne è uno scambio diretto. Migrare ad Antigravity ti tiene nel mondo di Google; adottare il reviewer di Copilot, CodeRabbit, Qodo o Greptile ti restituisce le revisioni delle PR. In un modo o nell'altro, oggi è una buona giornata per collegare il sostituto prima che la prossima pull request resti senza revisione.
