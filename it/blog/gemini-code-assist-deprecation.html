Il reviewer GitHub di Gemini Code Assist si spegne oggi — dove andare adesso

17 luglio 2026
Gemini Code Assist AI Code Review Strumenti per sviluppatori GitHub

Se lasci che Gemini riveda le tue pull request, oggi è il giorno in cui il bot ammutolisce. A partire dal 17 luglio 2026, Google ha spento la versione consumer di Gemini Code Assist su GitHub — l'app che pubblicava revisioni del codice e riepiloghi automatici sulle tue PR. È l'ultimo passo di una dismissione a tappe dei piani gratuito e individuale in corso dalla primavera.

Ecco la sequenza completa, perché i pezzi si sono spenti in date diverse:

  • 18 giugno 2026 — le estensioni IDE di Gemini Code Assist e la Gemini CLI hanno smesso di servire richieste per Gemini Code Assist per privati e per i piani Google AI Pro e Ultra.
  • 17 luglio 2026 (oggi) — l'app GitHub consumer di Gemini Code Assist cessa la sua attività di revisione del codice.

Una precisazione importante prima che qualcuno vada nel panico: questa è la storia consumer. Se la tua organizzazione paga una licenza Gemini Code Assist Standard o Enterprise, Google dice che quelle sono non interessate — l'assist nell'IDE, la CLI e le revisioni su GitHub continuano a funzionare. Ciò che finisce è la porta d'accesso gratuita/individuale che molti sviluppatori solitari e piccoli team avevano collegato ai propri repository.

Cosa Google vuole che tu faccia

Il percorso di migrazione proposto da Google è Antigravity, la piattaforma «agent-first» presentata all'I/O di maggio, più il suo compagno da terminale, Antigravity CLI — una riscrittura in Go che condivide lo stesso agent harness dell'app desktop Antigravity 2.0. È disponibile dal 19 maggio su antigravity.google/download, e i tuoi Agent Skills, Hooks, Subagents ed Extensions (ora «plugin») vengono trasferiti. Google è chiara sul fatto che al lancio non c'è piena parità di funzionalità.

Ma nota il divario. Antigravity CLI è un agente che guidi tu — un assistente di coding interattivo nel tuo terminale e nel tuo editor. La cosa che oggi si è effettivamente spenta era un reviewer automatico che sorvegliava le tue PR su GitHub e le commentava senza che glielo si chiedesse. Non sono lo stesso lavoro, e Google non sta rilasciando un sostituto identico per il bot GitHub consumer. Quindi se ciò che ti mancherà sono le revisioni delle PR, la risposta onesta è: probabilmente ti serve uno strumento dedicato di code review, non solo Antigravity.

Con cosa l'ho sostituito

Alla fine ho configurato una piccola coda di revisione indipendente dal fornitore per 31 dei miei repository; il 32º repository protetto è pronto in una pull request con tutti i controlli superati. Un workflow GitHub Actions riconosce la paternità di Claude dal nome del branch, dal corpo della PR o dai metadati di autore e sessione dei commit. Quando una di queste pull request viene aperta, aggiornata, riaperta o contrassegnata come pronta per la revisione, aggiunge l'etichetta codex-review. Codex prende in carico la coda, legge l'intero diff e i thread di revisione irrisolti, controlla la CI esistente, esegue i test locali pertinenti e pubblica una revisione supportata da evidenze oppure invia una correzione strettamente circoscritta. Al termine di una revisione riuscita, l'etichetta viene sostituita da codex-reviewed; un nuovo commit annulla quello stato e rimette la PR in coda. Il merge resta una decisione separata ed esplicita.

Richiede più configurazione rispetto all'installazione di una singola GitHub App, ma conserva ciò che per me contava davvero: revisione automatica a ogni modifica, stato visibile, controlli riproducibili e nessuna presunzione che l'agente che scrive il codice debba esserne anche il revisore finale.

Quanto era valido Gemini, in concreto?

Prima di sostituire l'app ho esaminato 10 pull request. Mentre completavo questo articolo nella data prevista per lo spegnimento, Gemini ha revisionato anche questa PR e aggiunto 14 osservazioni valide su localizzazione e SEO. Nelle 11 revisioni ha pubblicato 54 osservazioni inline: 5 ad alta priorità, 1 sicurezza-media e 48 media. Ne ho giudicate 53 utilizzabili e una un falso positivo, pari a un tasso di utilità del 98,1% per commento. Raggruppando le ripetizioni — come un campo di metadati mancante in sei traduzioni — corrispondono a circa 31 schemi di problemi distinti. Quest'ultima revisione mostra che il taglio non è stato istantaneo, anche se la data di spegnimento consumer pubblicata da Google restava il 17 luglio.

I risultati utili erano concreti: gestione degli errori mancante, modifica non sicura dell'archivio certificati, presupposti errati su Unicode e locale, configurazione systemd non portabile, HTML malformato, percorsi di pulizia che fallivano in caso di eccezione e prerequisiti degli strumenti mancanti. Gemini era particolarmente efficace sui difetti locali con una correzione chiara. Lo era meno nel consolidare i duplicati e nel ragionare sull'intero workflow: diversi commenti ripetevano la stessa causa in file o lingue differenti, una volta ha scambiato il JSON di un notebook per una tupla Python e non ha individuato una race condition di idempotenza scoperta da una seconda revisione. Il mio giudizio è favorevole ma circoscritto: il bot ha prodotto molto valore reale, ma i suoi commenti richiedevano comunque verifica e un revisore capace di guardare oltre le singole righe.

La conclusione

Google non ha ucciso lo sviluppo assistito dall'IA; lo ha consolidato. Gli strumenti consumer di Gemini Code Assist vengono confluiti in Antigravity, e le licenze enterprise proseguono invariate. Ma la comodità specifica su cui molti di noi contavano — un reviewer gratuito che commentava discretamente ogni PR — da oggi è sparita, e Antigravity non ne è uno scambio diretto. Migrare ad Antigravity ti tiene nel mondo di Google; io ho preso la direzione opposta e ho affidato la revisione delle PR a Codex, così ogni modifica continua a essere controllata. In un modo o nell'altro, oggi è una buona giornata per collegare il sostituto prima che la prossima pull request resti senza revisione.
